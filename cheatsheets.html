01

Introduction

Hypertext Markup Language (HTML) and Cascading Style Sheets (CSS) can be paired together to create a web design.

HTML controls the structure, content, words and CSS controls the colours, styles, and sizes.

HTML

HTML documents are composed of HTML elements.

All HTML pages start with a doctype declaration that tells the browser which version of HTML is being used. The latest version is HTML5.

<!DOCTYPE html>

After the doctype declaration, HTML pages then have an html element.

<html></html>

Inside the html element is a head element containing page info followed by a body element containing page content.

<head></head>
<body></body>

Most HTML elements have opening and closing tags.

<p>An example of opening and closing tags</p>

HTML elements can have attributes that add more information.

<a href="www.example.com"> Example </a>

Some HTML elements do not use a closing tag.

<img src="example.png">
CSS

CSS uses rules to control the presentation of HTML elements. CSS rules have a selector, properties and values.

p {
 color:pink;
 font-size:14px;
}

A selector is used to select which elements a rule applies to. In the above example, p will apply this rule to HTML <p> elements.

The selector is followed by curly braces which define where the details of the rule begin and end.

A property defines which aspect of an element is being controlled. In the above example, color will control the <p> element text colour and font size.

A value defines the value that a particular property will take. In the above example, text colour will be pink and font size will be 14 pixels.

CSS can be added to a webpage using inline styles, internal css, or external stylesheets.

Inline styles add the style attribute to HTML elements.

<p style="color:pink">Example</p>

Internal CSS puts CSS rules inside a <style> element placed in the head section of a webpage.

<head>
 <style>
  p {
   color:pink
  }
 </style>
</head>

External stylesheets use a link element in the head section that links to an external .css file containing CSS rules.

*Important*
Using external stylesheets is the recommended way to add styles to a webpage.

<head>
 <link rel="stylesheet" type="text/css" href="main.css">
</head>

02

Links, Colours & Fonts

Links

Links are created using the anchor tag and the href attribute (Hypertext Reference) defines a link's destination.

<a href="www.example.com">Example Website</a>

The href can specify a relative path for local content, an absolute path to a different webpage, or an element on the same page.

<a href="/about.html">Example Relative Path</a>

<a href="www.example.com">Example Absolute Path</a>

<a href="#Section_lower_down">Example Jump to Section 2</a>
Next section: <h2 id="#Section_lower_down">Section 2</a>

Links can also lead to email addresses and phone numbers.

Anchor tags can have target and title attributes.

The target attribute tells the browser where to open the link and the title attribute displays a tooltip when the user hovers over a link.

<a href="/monstera deliciosa.jpg" target="_self" title="Photo of a monstera deliciosa">Houseplant</a>

Links can be in different states depending on user interaction:

  • unvisited
  • visited
  • hover
  • active
Colours

The RGB colour model is used to display colours on screens using red, green and blue light.

Colours can be referred to using:

  • colour names
  • RGB values
  • hex codes
  • HSL values

All browsers recognise 16 basic colour names while modern browsers recognise an extended list of 100+ color names.

RGB values specify the red, green, and blue component of a colour with a number from 0-255.

rgb(238,88,159)

Hex codes use pairs of hexadecimal symbols to specify the red, green, and blue components with values from 00-FF.

#ee589f

Using an alpha value to update a colour's transparency will change the hex code format from #RRGGBB to #RRGGBBAA (where alpha is A).

#ee589f80

HSL values specify the hue (0-360°), % saturation, and % lightness of a colour.

hsl(332,82%,64%)
Styling Text

Some of the main CSS properties that control font formatting and styling are:

  • color
  • font-weight, font-style, text-decoration
  • font-family
  • font-size

The color property controls text colour.

For basic text formatting, we use:

  • font-weight: bold
  • font-style: italic
  • text-decoration: underline

The font-family property can specify a generic family like serif, sans-serif, or monospace, or a specific font.

A font stack lists multiple fonts one after the other to ensure fonts load as desired.

The font-size property can use absolute units (px, pt, xx-small through xx-large) or relative units (em, %, smaller, larger).

Absolute units remain consistent while relative units set the font size in relation to the parent element.

03

Lists, Tables & Images

Lists

Lists contain groups of related data.

Ordered lists are created using <ol></ol> tags.

Unordered lists are created using <ul></ul> tags.

No matter the list type, each list item is enclosed in <li></li> tags.

The CSS rule list-style-type controls the bullet or number style used by a list.

For unordered lists, list-style-types include disc, circle, and square.

For ordered lists, list-style-types include decimal, lower-latin, upper-latin, lower-roman, and upper-roman.

Tables

HTML tables are created using the <table> opening and closing tags.

For each row in the table, add a pair of table row tags, <tr></tr>.

Use the table cell data tags <td></td> to add cells to each row.

For header cells, use table cell header tags <th></th> which most browser styles default to bold.

Table rows can be grouped together into header, body, and footer sections.

To do this, nest the rows in <thead>, <tbody>, and <tfoot> opening and closing tags.

Images

Images are embedded in a page using an empty <img> tag.

The src and alt attributes set the location of the image file and alternative text to describe the image.

<img src="/twitter.png" alt="Twitter icon">

Relative paths can point to the current directory, a parent folder, or multiple levels up.

<img src="images/example.jpg">
<img src="../images/example.jpg">
<img src="../../images/example.jpg">

Images can be styled by changing the height, width, border, padding, opacity, and border-radius.

The default size of an image element is determined by the size of the image itself.

You can resize an image with the CSS height and width properties

img {
  width: 200px;
  height: 100px;
}

You can make the image look transparent using the opacity property.

Opacity is expressed as a number between 0 and 1 where 0 is 0% opaque and 1 is 100% opaque.

img {
  opacity: 0.5;
}

Images can be wrapped inside anchor tags to create a link.

<a href="https://twitter.com">
  <img src="/twitter.png" alt="Twitter icon">
</a>

You can add padding to create empty space between the image and the border.

img {
  border: 2px solid pink;
  padding: 25px;
}

You can round the corners of an image by setting a border-radius.

A single value will set the border-radius for all four corners while two or four values specify the radius for different corners.

The following example defines the border radius values for the top left, top right, bottom right, bottom left:

img {
  border-radius: 20px 20px 0px 0px;
}

Create completely round edges by setting border-radius equal to 50% of the image size:

img {
  border-radius: 50%;
}
Borders

Borders can have a border-width, border-style, and border-color.

Border-width determines the thickness of the border and can be measured in:

  • absolute units like px, pt
  • relative units like em, ext
  • thin, medium or thick

Border-width values are arranged top, right, bottom, left so remember the acronym TRouBLe!

A border-width with two values specifies top/bottom and left/right.

p {
  border-width: 10px 20px;
}

A border-width with three values specifies top, left/right, and bottom.

p {
  border-width: 10px 5px 20px;
}

A border-width with four values specifies top, right, bottom, and left (TRBL).

p {
  border-width: 10px 5px 20px 5px;
}

The border-style determines the line pattern of the border.

Common Styles Example
solid ______
dashed _ _ _ _
dotted . . . . . .
double ===== (unbroken irl)

3D Styles Description
inset engraved
outset embossed
groove etched
ridge raised

As with border-width, border-style values are arranged TRBL and a combination of one, two, three, or four values may be specified.

p {
  border-style: solid dashed dotted double;
}

The colour of the border can be set with the border-color property.

Border colours can be specified in the same way as other CSS colours such as by name, hex, rgb or hsl value.

As with border-width and border-style, the border-color values are arranged TRBL and a combination of one, two, three, or four values may be specified.

The border property can be used as a shortcut to style border-width, border-style and border-color.

p {
  border: 5px dotted lemonchiffon;
}

Shortcuts can also style each side of the border independently:

p {
  border-top: 5px solid mistyrose;
  border-right: 2px dotted lemonchiffon;
  border-bottom: 5px solid mistyrose;
  border-left: 2px dotted lemonchiffon;
}
CSS Box Model

The CSS box model describes how an HTML element's content, padding, border and margin fit together and the space the element occupies on the page.

At the center of the box are the contents and you can control the size of these contents using the CSS height and width properties.

The CSS padding property sets a space between an element's content and its border.

The default padding value varies by element and browser.

You can specify one value for consistent padding all around, two values to set top/bottom and left/right, or four values for every side (TRBL).

img {
  padding: 5px 10px 5px 10px;
}

The border wraps around the element's content and padding.

img {
  border: 5px dotted lemonchiffon;
  padding: 5px 10px 5px 10px;
}

The margin wraps around the element's border and determines spacing between this element and other elements.

Margin can be set on each side with margin-top, margin-right, margin-bottom, and margin-left, or by using the shorthand margin followed by four values (TRBL).

img {
  border: 5px dotted lemonchiffon;
  padding: 5px 10px 5px 10px;
  margin: 5px 10px 5px 10px;
}

So, the total space occupied by an element is its content height/width, padding, borders, and margins. This is the box model!

04

Divs, Spans, IDs & Classes

HTML Formatting Tags

While most HTML elements are designed to add meaning rather than add presentation, there are still some tags that are used primarily for formatting.

<sub> for subscript text like this
<sup> for superscript text like this
<del> for deleted text
<ins> for inserted text

The strong and emphasis HTML elements indicate meaning as well as formatting.

<em> for emphasis, italic default
<strong> for important text, bold default

The default styling for <em> and <strong> can be adjusted without affecting the meaning of the elements.

Note: Modern web pages have moved away from using <b>, <i>, and <u> tags.

Divs & Spans

Div and span are generic elements without semantic meanings but are used to help style HTML.

The <div> is a generic division element used to define containers or sections within our HTML.

Wrapping a group of elements within a <div> is most useful when styling in CSS.

Div elements are block level elements, which means that by default:

  • Div elements start on a new line
  • Elements after a div start on the next line
  • They take up the full width of their parent element

The <span> element is used to style individual sections within elements, making it easier to add a little formatting.

Span elements are inline elements, which means that by default:

  • Span elements do not start on a new line
  • Content after a span element continues on the same line
  • They are only as wide as their contents.
CSS Display Property

The CSS display property controls whether an element appears as an inline or block level element or whether an element is rendered on the page at all.

Some possible values of the display property are:

  • block
  • inline
  • inline-block
  • none

A block level element:

  • starts on a new line
  • takes up the full width of its parent element
  • has elements after it start on the next line
  • allows width and height to be controlled using CSS properties
p {
  display:block;
  height:25px;
  width:25px;
}

Example block level elements include <h1>, <p>, and <div>.

An inline element:

  • does not have its own line
  • is the width of its contents
  • does not allow width and height to be adjusted
p {
  display:inline;
}

Example inline elements include <a>, <em>, and <span>.

An inline-block element:

  • does not have its own line
  • is the width of its contents
  • allows width and height to be controlled using CSS properties
p {
  display:inline-block;
  height:25px;
  width:25px;
}

The <img> element has a default display value of inline but acts as if it has the styling inline-block.

An element with a display property of none is removed from the layout and is not rendered.

p {
  display:none;
}
IDs & Classes

An ID identifies an element on a page. They are unique and can only be placed once on a page.

We style an element with a particular ID using the CSS selector # followed by the ID name.

#introParagraph {
  color: blue;
}

We can add an ID attribute to any HTML element to uniquely identify it.

<p id="mainStudy"> Machine Learning is a subset of AI.</p>

#mainStudy {
  font-size: 15px;
}

A class attribute identifies similar elements on a page. They can be added in multiples.

<ul>
  <li class="language"> HTML</li>
  <li class="language"> CSS</li>
</ul>

We style elements of the same class using the CSS selector . followed by the class name.

.language {
  color: blue;
}
Background Styling

It can be useful to change the background styling of an element, especially block level or inline, to identify the space it is taking up on the page.

There are several CSS background properties we can control:

  • background-color
  • background-image
  • background-repeat
  • background-position

The background-color property uses the same rules as color and border-color properties.

div {
  background-color: hotpink;
}

The background-image property can be set on any element using the url value followed by the image location:

div {
  background-image: url('/hearts.jpg');
}

The background-repeat property controls how a background image will repeat to fill the element it's set on.

div {
  background-image: url('/hearts.jpg');
  background-repeat: repeat;
}

value description
repeat default, repeats both directions
repeat-x repeats horizontally
repeat-y repeats vertically
no-repeat does not repeat in any direction

The background-position property uses the values left or right and top or bottom to place the image relative to the element.

div {
  background-image: url('/hearts.jpg');
  background-repeat: no-repeat;
  background-position: left top;
}

To keep our CSS neater, we can use the background property as a shorthand for the other properties:

  1. background-color
  2. background-image
  3. background-repeat
  4. background-position

div {
  background: hotpink url('classroom.jpg') no-repeat right bottom;
}

CSS Selectors & Positioning

There are additional CSS selectors beyond IDs and classes.

The position of elements on a page can be altered using the CSS position property.

Descendent Selectors

The descendent selector pattern is used to style any element that's within another element.

Descendent selectors are marked in CSS by having a space between two selectors:

ancestor descendant {
  property: value;
}

The following descendent selector styles only those <p> elements that are descendents of <div> elements.

div p {
  background-color: yellow;
}
Child Selectors

The child selector pattern styles only elements that are direct children of the first element specified.

In other words, child selectors affect styling only one level deep in your markup.

Child selectors are marked in CSS by having a > between two selectors:

parent > child {
  property: value;
}

The following child selector styles only those <p> elements that are direct children of <div> elements.

div > p {
  background-color: yellow;
}
Combining Selectors

We target elements with a particular class or ID using the CSS selectors:

.class { }
#id { }

We can use the same child and descendant selector patterns with CSS classes and IDs.

.wrapper .content {
  background-color: yellow;
}

The above descendent selector will give a yellow background to every element with the content class inside an element with the wrapper class.

#main > .content {
  background-color: yellow;
}

The above child selector will give a yellow background to only first-level .column elements within the #main element.

If there were .column elements that were nested deeper then they would not be selected for styling.

Note: the selecting rules apply no matter how many levels deep or how specific you want to be .. but it's best to avoid overly specific selectors!

Static Positioning

Static positioning means not moving the element at all from its usual position on the page.

If no position is specified in CSS, the default styling for all elements is:

position: static
Relative Positioning

We use the top, left, bottom, and right properties to control how far we displace an element.

position: relative;
left: 5px;

The above example moves the left edge of an element 5px from its usual position in the positive direction (ie. to the right).

position: relative;
left: -5px;

The above example moves the left edge of an element 5px from its usual position in the negative direction (ie. to the left).

position: relative;
top: 5px;

The above example moves the top edge of an element 5px from its usual position in the positive direction (ie. down).

position: relative;
top: -5px;

The above example moves the top edge of an element 5px from its usual position in the negative direction (ie. up).

Absolute Positioning

Absolute positioning takes the element out of its usual position on the page.

If no other elements on the page have a position set, absolute positioning places an element relative to the browser window.

position: absolute;
top: 0px;
left: 0px;

The above example positions an element in the top left corner of the page.

The layout of any other elements on the page is as thought the absolutely positioned element isn't there.

If you absolutely position an element and any of its ancestor elements have a position set (other than static)..

div {
  position: relative;
}

#p {
  position: absolute;
  top: 0px;
}

..then the absolute positioning will place the element relative to its closest ancestor with a position set.

Fixed Positioning

The style position: fixed will position an element relative to the browser window.

#sticky {
  position: fixed;
  top: 100px;
  right: 100px;
}

It will keep that element fixed in position even as the browser window scrolls.

Modern HTML & CSS

HTML5 Semantics

HTML5 is the latest version of HTML adding new capabalities such as elements used for multimedia.

It also includes new semantic elements which better describe the meaning of the content on the page.

It's important to be clear and descriptive, mostly for structure and accessibility.

Using elements for their given purpose makes it easier for both people and machines to read and understand your code.

HTML5 semantics allow us to identify specific content elements in our markup such as:

  • header content
  • site navigation
  • standalone sections
  • self-contained articles
  • footer content

The <header> element can be used to represent your site's main header.

It includes content that is used across all pages of your website, such as a website logo.

The <footer> element is designed to enclose your website's footer section.

In a footer, you would typically find website author content, copyright information, social links, or even a sitemap.

The <nav> element can wrap a site's navigational links.

It usually sits at the top, left, or right of the main content and might be wrapped in the <header> element.

The <section> element represents a standalone section of content on your website.

It is usually used to contain and organize repetitive groups of content together.

A section element should include a h1-h6 title to identify the content within it.

The <article> element identifies self-contained and independent content.

Typically, article elements are used to contain blog posts, news articles, and stories.

CSS Flexbox

CSS Flexbox is a set of CSS rules that provides a more efficient way to layout, align, and distribute space and items in a container.

By default, block elements are placed vertically down the page.

When we use Flexbox, we can design more complex web page designs by challenging the default block and inline layouts.

To start a Flexbox layout, all you need is a container (parent) with children in it.

Changing the display property on that container enables flexible content for all its direct children.

Flexbox works in two axes: horizontal (justify) and vertical (align).

The flex-direction sets how the flex-items should flow.

By default, the direction is horizontal and would be referred to as the main axis.

The cross axis is perpendicular, vertical by default.

Display-Flex

To introduce a flexbox layout, you change the display property on the parent element which is wrapping your layout.

.container {
  display: flex;
}

The above property and value enables a flex context for all the parent's direct children.

Flex-Direction

Flex-direction sets the main axis to horizontal or vertical to control the flow of flex-items.

The default flex-direction value is row.

.container {
  display: flex;
  flex-direction: row;
}

The above style will place your flex items horizontally in a row from left to right.

The flex-direction row-reverse keeps the flex items horizonatlly in a row but reverses the order.

.container {
  display: flex;
  flex-direction: row-reverse;
}

Flex-items can also be placed in a column from top to bottom.

.container {
  display: flex;
  flex-direction: column;
}

The above style will place your flex items vertically in a column from top to bottom.

Lastly, the flex-direction column-reverse places items in a column but reverses the order from bottom to top.

.container {
  display: flex;
  flex-direction: column-reverse;
}
Flex-Wrap

Flexbox will squeeze everything onto one line unless you alter the flex-wrap property.

The default flex-wrap value is no wrap.

.container {
  display: flex;
  flex-wrap: nowrap;
}

The above style will mould all the flex-items into one row in your container.

When your content is looking too squashed, you can adjust teh flex-wrap value to wrap.

.container {
  display: flex;
  flex-wrap: wrap;
}

The content items will revert to their original widths and wrap onto multiple lines.

Lastly, the flex-wrap value wrap-reverse will wrap onto multiple lins from bottom to top.

.container {
  display: flex;
  flex-wrap: wrap-reverse;
}
Justify-Content

The justify-content property will distribute extra space that's available along your main axis.

The default value is flex-start which arranges flex-items at the start of the container.

.container {
  display: flex;
  justify-content: flex-start;
}

Flex-end will shift the space in your container and pack the flex-items at the end.

.container {
  display: flex;
  justify-content: flex-end;
}

To center items along your main axis, use the value center.

.container {
  display: flex;
  justify-content: center;
}

To have items evenly spaced from the start of your flex container to the end, use the value space-between.

.container {
  display: flex;
  justify-content: space-between;
}

To have items evenly dispersed with equal space around each item, us the value space-around.

.container {
  display: flex;
  justify-content: space-around;
}

Lastly, use space-evenly to have all available space evenly dispersed.

.container {
  display: flex;
  justify-content: space-evenly;
}
Align-Items

The align-items property will define how the flex items are laid out perpendicular to the main axis set.

With the align-items property remaining on the default value of stretch, the items fill the container perpendicular to the flex-direction.

.container {
  display: flex;
  align-items: stretch;
}

The value flex-start will take away the default stretching behavior and place the items at the start of the cross axis.

.container {
  display: flex;
  align-items: flex-start;
}

To place the items at the end of the cross axis, you would change the align-items property to flex-end.

.container {
  display: flex;
  align-items: flex-end;
}

Lastly, items can be aligned in the center of the cross axis with the center value.

.container {
  display: flex;
  align-items: center;
}
Align-Content

The align-content property manipulates the distribution of space between and around elements that run over multiple lines.

By default, align-content will stretch the content in your container to fill all the remaining space.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: stretch;
}

To set the content at the start of the container, set align-content to flex-start.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: flex-start;
}

Note: as with all flex properties, align-content will honour the flex-direction set on the container, too.

To set the content at the end of the container, use the value flex-end.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: flex-end;
}

To have the multi=line content centered, set the value to center.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: center;
}

The space-between value will evenly distribute the space between each line while keeping the first line at the start and the last at the end.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: space-between;
}

To have equal space on both sides of each line, use the space-around value.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: space-around;
}

Or, to have your lines evenly distributed in the remaining space, use the space-evenly value.

.container {
  display: flex;
  flex-wrap: wrap;
  align-contents: space-evenly;
}

Resolving CSS Conflicts

The CSS cascade and understanding how browsers resolve competing CSS rules.

Inheritance

In CSS, if we don't set the value of a property on an element, one of two things will happen depending on which property:

  1. The element will inherit the styling of that property from its parent element;
  2. The element will inherit the stylinging of the initial value of that property.

Many elements defined in your markup will inherit the styling of their parent elements.

However, there are some properties which have a default "do not inherit" behaviour.

These are typically properites that are going to dramatically change the way your element looks, eg. border, margin, padding, and width.

If we wanted these non-inherited properties to apply to the child elements, we would need to specifically seet them, too.

Positioning

As our stylesheets get larger, we might find ourselves writing CSS declarations that are not being displayed on our page .. and not understand why.

One of the most important rules for how our browsers resolve competing CSS style rules is the position in which these rules appear in our stylesheet.

The position of rules in your CSS matters when you have competing style rules.

For example, if you have declared a background-image on an element twice, or if you have set up two rulesets for a <div> element in your CSS..

.. then the order in which you declare these CSS rules matters.

Rules that are defined later in your stylesheet will override those that appear earlier.

This same "latest-declared" rule also applies if properties are applied twice within the same ruleset.

The last property and value specified will take precedence over those declared earlier in your stylesheet.

Specificity

We know that we can set IDs and classes on our HTML elements and then use those as selectors in our CSS.

There is a precedence set for which selectors are considered more specific than others.

IDs are the most specific.

Selectors which target IDs will override those that are using classes or direct elements.

Classes are less specific than IDs but more specific than HTML elements.

Classes are one of the most popular selectors to use because they can target particular elements and be applied to multiple elements at once.

A particular class can be set again and again on different elements throughout our HTML, allowing us to apply the same styles on similar elements.

A single element can also have multiple classes applied to it.

<h1 class="title right">

To be more specific in our CSS declaration, we can target multiple classes.

.title.right {
  color: navy;
}

In terms of what takes precedence in our stylesheets, multiple-class selectors rank higher than single-class selectors as they are more specific.

The least specific way to target CSS is by simply using the HTML element itself as a selector.

A CSS rule which targets the element using a class is more specific than a rule which targets the HTML element.

So, a rule with a class affecting an HTML element will override a rule without.

Importance

There is a special-case rule called !important which overrides any rule, no matter how specific.

.panel {
  color: red !important;
}

However, the !important rule should be treated with extreme caution.

If you set !important on rules because the position, specificity, and inheritance aren't being used correctly, your stylesheets will soon become difficult to read and maintain.

Reserve the use of !important for an emergency scenario where you have no other option.

Responsive Web Design

A responsive webpage can change according to the user's screen size, device, browser, and/or orientation.

A responsive design often uses relative units like percentages or ems for things like element width.

Viewport

The viewport is the size of the user's browser window.

A user's viewport can vary depending on the device or browser they're using.

We can control how a webpage should render within a user's viewport using the <meta> tag.

The meta tag sits within the head of our HTML page.

<meta name="viewport" content="width=device-width", initial-scale=1" />

The content attribute allows us to control the width and zoom level of the viewport.

Media Queries

A media query is a CSS feature that allows us to apply CSS styles only when a user browses a website with certai browser viewport sizes, devices, media, or device orientations.

Media queries can include media types (optional), media feature expressions, and logical operators.

We write a media query as follows:

@media only screen and (max-width: 480px) {
  h1 {
    font-size: 16px;
  }
}

The and operator allows us to combine more than one media type or feature in a media query.

The media feature expression tells the browser when to apply these styles.

There are several media types available including screen, print, speech, and all.

We can combine conditions in our media queries with operators such as and, only, and not only.

We can also style for different deivce orienations: portrait or landscape.

For example, we could change the direction of a flexbox layout for potrait and landscape orientations.

@media (orientation: portrat) {
.container {
  flex-direction: column;
 }
}
@media (orientation: landscape) {
.container {
  flex-direction: row;
 }
}

The points where we change our styling with media queries are called breakpoints.

It is most common to pick breakpoints to target devices of certain widths to address specific styling issues.

Mobile-first Design

In mobile-first design, we create our styling with users on mobile devices first.

We then add adjustments to our styling such as media queries to target users on other devices like desktop computers or tablets.

Instead of leaving out information to fit smaller devices, developers adapt the content to fit any device.

It's more about progressively enhancing the content as a user's viewport increases.

Mobile-first design considers things such as:

  • Simplicity
  • Navigation that is easily used on a small screen
  • Links large enough to be used by fingertips
  • Optimising for slower internet connections

When developing with a mobile-first approach, we should set up our CSS rules for mobile as a foundation and then beuild on that using things like media queries.

HTML Forms

An HTML form is a set of inputs that allow a user to enter data into a form on the webpage.

Once the user submits the form, their responses are sent to a server or another page.

We can insert a form into our webpage using the <form> element.

There are many different kinds of form fields depending on the desired user interaction and data.

The Input Element

The input element is the most common form field with a considerable list of capabilities.

By altering the type attribute, the input field becomes quite powerful.

<form>
  <input type="text" name="name">
</form>

A single-line text field is the default HTML input element.

Changing the type attribute will alter the input appearance and behaviour.

<form>
  <input type="text" name="name" placeholder="Full Name">
  <input type="email" name="email" required>
  <input type="submit" value="Submit">
</form>

The list of available attributes for the input element is long and diverse.

Depending on which attributes are set, it will affect the interaction your user has with the input.

Labels

The <label> element is useful for associating an input with a caption or direction for how to complete the field.

<label>
  What's your name?
  <input name="full-name">
</label>

Or, the label can be defined separately and then associated with the input via a shared attribute.

<form>
  <label for="inspo-person">
  Who inspires you?
  </label>
  <input type="text" id="inspo-person">
</form>

Adding a label to your input also gives the user the ability to click on the label to auto-focus on the associated input.

Both nested and separate formats will associate the two elements leading to better UX and accessibility.

Textareas

The <textarea> element can be used for multi-line text responses.

<textarea name="message">
  Message..
</textarea>

Text areas have similar features to text inputs though the markup is slightly different.

Textareas do not support the value attribute.

Instead, the default value is wrapped in opening and closing textarea tags.

Multiple-choice Fields

There are a few form fields we can use if we want to give the user a choice between predefined options.

Let's look at:

  • dropdowns
  • checkboxes
  • radio buttons

Dropdowns are created using the <select> element.

<select name=select-time">
  <option value="am">AM</option>
  <option value="pm">PM</option>
</select>

A select element has opening and closing tags with a list of options nested in between.

Each option should have a value attribute to set the value of the user's selection to send to the server when the form is submitted.

Checkboxes and radio buttons also allow a user to select from a predefined list of options.

We use the <input> element for both but with different type attributes set.

To create a checkbox we use an input element and set the type attribute to checkbox.

<input type="checkbox"
 name="gang" id="gang" value="Join">
 <label for="gang">Join the girl gang?</label>

For a radio button, we use the input element and set the type attribute to radio.

<input type="radio"
 name="when" id="week" value="weekly">
 <label for="week">Weekly</label>

The important distinction between these two input types is that multiple checkboxes can be selected at once.

However, radio buttons should be grouped into a set.

To do this we declare the same name for each input in the set.

Users can only select one of the radio buttons in a set at a time.

For good accessibilty, checkbox and radio button input types should have associated labels declared alongside them.

Form Submission

At the end of a form, we can display a button for a user to click on once they've finished filling out the form.

Clicking the button submits the user's responses to a server or endpoint for processing.

To display a submit button we can use an input element with a type attribute of submit.

<input type="submit" value="Send">

The submit input type is displayed as a button that a user can click.

The text displayed is defined via the value attribute.

There are several attributes on the form element which control the way a form is submitted.

The two most important attributes are the action and method attributes.

The action takes care of where form data is sent while the method defines how.

<form method="get" action="send.php">
...
</form>

The action attribute usually points to a webpage or URL that has been designed to process the form data we're sending.

The method attribute controls what type of request is used to send the data.

The value of the method can be either get or post.

This is because get and post are the two types of HTTP web requests we can send.

HTTP (Hypertext Transfer Protocol) is a communications protocol between the client (user device) and server (the machine/s where backend software is hosted).

To send data via HTTP we send a request.

When we submit an HTML form, we tell our webpage to send the data as an HTTP request.

There are two types of request we can use to send our form data:

<form method="GET">

A get request is usually used to retrieve data from the server and is smaller than a post request.

<form method="POST">

A post request is used to post data to a server and usually involves sending a lot of data.
